# 不採用方式アーカイブ

設計過程で検討・不採用とした方式のアーカイブ。現行設計の理由は `docs/design.md` を参照。

## send-keys: クリップボード+ペースト方式（不採用）

テキストをpbcopyでクリップボードにセットし、Cmd+Vでペーストする方式。設定変更不要というメリットがあるが、以下の理由で不採用とした:

- **クリップボード汚染**: ユーザのクリップボード内容が一時的に上書きされる（退避・復元は不完全）
- **秘密情報漏洩**: クリップボードマネージャ（Maccy, Alfred, Raycast等）がsend-keysの内容を履歴に記録する可能性
- **排他制御の複雑さ**: 複数gmuxプロセスの同時send-keysにflock排他制御が必要
- **フォーカス移動の必要性**: ターゲットペインへのフォーカス移動が必要で、send-keysの副作用でペインフォーカスが変わる

### 比較表

| 観点 | PTYプロキシ方式（採用） | クリップボード方式（不採用） |
|------|---------------|------------------------|
| クリップボード汚染 | なし | あり（退避・復元で軽減するが完全ではない） |
| 秘密情報漏洩リスク | なし | クリップボードマネージャに記録される可能性 |
| 排他制御 | 不要 | flock排他制御が必要 |
| フォーカス移動 | 不要（ソケットでペイン指定） | 必要（ターゲットペインにフォーカス移動が必要） |
| 特殊キー送信 | エスケープシーケンスとして直接書き込み | keystrokeシミュレーション |
| 設定変更 | ghostty config: `command = gmux-agent` | 不要 |

## send-keys: キーシミュレーション方式（不採用）

System Eventsで1文字ずつkeystrokeを送る方式。遅く不正確。

## send-keys: PTYスレーブ直接書き込み（不可能）

PTYスレーブへの直接書き込みは画面表示のみでシェル入力にならないことが検証で確定した。

## send-keys: TIOCSTI ioctl（不可能）

macOS Ventura以降でブロックされている。
